# 📦 Руководство по сборке Wavify

## Подготовка к сборке

### 1. Установка зависимостей серверов

Перед сборкой нужно установить зависимости для всех серверов:

**Windows:**
```bash
cd desktop
install-server-deps.bat
```

**Linux/Mac:**
```bash
cd desktop
chmod +x install-server-deps.sh
./install-server-deps.sh
```

Или через npm:
```bash
cd desktop
npm run install-deps
```

### 2. Проверка конфигурации

Убедитесь что все конфигурационные файлы на месте:

- `backend/.env` - настройки backend сервера
- `youtube-audio-server/yandex_token.txt` - токен Яндекс.Музыки (опционально)
- `youtube-audio-server/www.youtube.com_cookies.txt` - куки YouTube (опционально)
- `telegram-bot/.env` - токен Telegram бота

## Сборка приложения

### Windows

```bash
cd desktop
npm run build
```

Готовое приложение будет в папке `desktop/dist/`

### Mac

```bash
cd desktop
npm run build:mac
```

### Linux

```bash
cd desktop
npm run build:linux
```

## Что включается в сборку

При сборке автоматически включаются:

1. **Desktop приложение** - основной интерфейс
2. **Backend сервер** (порт 3001) - API для работы с данными
3. **YouTube Audio Server** (порт 3003) - стриминг музыки
4. **Telegram Bot** (порт 3004) - авторизация через Telegram

Все серверы запускаются автоматически при старте приложения.

## Автозапуск серверов

При запуске собранного приложения:

1. ✅ Автоматически запускается Backend сервер (порт 3001)
2. ✅ Автоматически запускается YouTube Audio Server (порт 3003)
3. ✅ Автоматически запускается Telegram Bot (порт 3004)

Если порты заняты, серверы не запустятся и приложение покажет ошибку.

## Проверка работы серверов

После запуска собранного приложения проверьте:

1. **Backend API**: http://localhost:3001/health
2. **YouTube Server**: http://localhost:3003/health (если есть)
3. **Telegram Bot**: Проверьте логи в консоли

## Устранение проблем

### Серверы не запускаются

1. Проверьте что порты 3001, 3003, 3004 свободны
2. Проверьте логи в консоли (F12 в приложении)
3. Убедитесь что все зависимости установлены

### Telegram авторизация не работает

1. Проверьте что Telegram Bot запущен (логи в консоли)
2. Проверьте токен в `telegram-bot/.env`
3. Проверьте что порт 3004 доступен

### YouTube/Yandex не работает

1. Проверьте что YouTube Audio Server запущен
2. Проверьте куки и токены
3. Проверьте что порт 3003 доступен

## Размер сборки

Примерный размер готового приложения:

- **Windows (NSIS installer)**: ~150-200 MB
- **Mac (DMG)**: ~150-200 MB
- **Linux (AppImage)**: ~150-200 MB

## Оптимизация размера

Для уменьшения размера можно:

1. Удалить неиспользуемые зависимости
2. Использовать `--production` при установке npm пакетов
3. Исключить dev зависимости из сборки

## Дополнительные команды

```bash
# Сборка с автоматической установкой зависимостей
npm run build:full

# Только установка зависимостей серверов
npm run install-deps

# Запуск в dev режиме
npm run dev

# Обычный запуск
npm start
```

## Примечания

- При первом запуске собранного приложения серверы могут запускаться дольше обычного
- Все данные пользователя сохраняются в локальной базе данных
- Логи серверов доступны в консоли разработчика (F12)
