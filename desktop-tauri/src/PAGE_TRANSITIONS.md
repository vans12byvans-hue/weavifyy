# Анимации переходов между страницами

## Реализованные анимации

### 🎬 Переходы страниц

#### 1. **Основная анимация (fadeInUp)**
- Страница появляется снизу с плавным затуханием
- Длительность: 0.4s
- Easing: cubic-bezier(0.16, 1, 0.3, 1) - плавная пружинистая анимация

#### 2. **Анимация исчезновения (fadeOut)**
- Страница исчезает вверх с затуханием
- Длительность: 0.2s
- Easing: ease-out

#### 3. **Скольжение (slideIn)**
- Используется для страниц поиска и плейлистов
- Страница появляется справа
- Длительность: 0.4s

#### 4. **Масштабирование (scaleIn)**
- Используется для модальных страниц (настройки, профиль)
- Страница увеличивается от 95% до 100%
- Длительность: 0.3s

### 🎯 Анимации кнопок сайдбара

#### Hover эффект
- Кнопка поднимается на 2px
- Масштабируется до 105%
- Появляется тень
- Иконка поворачивается на 5° и увеличивается

#### Active состояние
- Анимация пульсации (buttonPulse)
- Иконка "подпрыгивает" с поворотом (iconBounce)
- Длительность: 0.4-0.6s

#### Click эффект
- Кнопка сжимается до 98%
- Быстрая анимация (0.1s)

### 📦 Анимации контента

#### Появление контента
- Все элементы внутри страницы появляются с задержкой 0.1s
- Плавное затухание + движение вверх
- Длительность: 0.5s

#### Последовательное появление элементов
- Треки, карточки альбомов и артистов появляются по очереди
- Задержка между элементами: 0.05s
- Максимум 5 элементов с задержкой (остальные появляются сразу)

## Технические детали

### Timing функции
```css
cubic-bezier(0.16, 1, 0.3, 1) - Плавная пружинистая анимация
ease-out - Быстрое начало, медленный конец
```

### Оптимизация производительности
- Используется `will-change` для анимируемых элементов
- GPU ускорение через `transform` вместо `top/left`
- Минимальное использование `opacity` для плавности

### Логика переключения
1. Клик по кнопке сайдбара
2. Текущая страница исчезает (200ms)
3. Задержка 200ms
4. Новая страница появляется (400ms)
5. Контент внутри страницы появляется с задержкой (100ms)

## Кастомизация

### Изменить скорость анимаций
```css
/* В styles.css */
.page.active {
  animation-duration: 0.6s; /* Медленнее */
}
```

### Отключить анимации
```css
.page {
  animation: none !important;
}

.page.active {
  animation: none !important;
}
```

### Добавить свою анимацию
```css
@keyframes myCustomAnimation {
  from {
    opacity: 0;
    transform: rotateY(90deg);
  }
  to {
    opacity: 1;
    transform: rotateY(0);
  }
}

.page.custom-style {
  animation: myCustomAnimation 0.5s ease-out;
}
```

```javascript
// В renderer.js
if (page === 'mypage') {
  pages[page].classList.add('custom-style');
  setTimeout(() => pages[page].classList.remove('custom-style'), 500);
}
```

## Производительность

### Рекомендации для слабых ПК
1. Уменьшить длительность анимаций до 0.2s
2. Отключить анимации контента
3. Убрать последовательное появление элементов

### Метрики
- FPS во время анимации: 60fps (на современных ПК)
- Задержка перехода: ~400ms
- Общее время перехода: ~600ms
