# Исправление проблемы с иконкой приложения

## Проблема
После установки приложения иконка меняется на стандартную иконку Electron вместо нашей кастомной.

## Что было исправлено

### 1. Конфигурация package.json
Добавлены дополнительные параметры для NSIS установщика:
```json
"nsis": {
  "installerIcon": "assets/icon.ico",
  "uninstallerIcon": "assets/icon.ico",
  "installerHeaderIcon": "assets/icon.ico"
}
```

### 2. Явное включение иконок в сборку
Добавлены иконки в список файлов для включения:
```json
"files": [
  "assets/icon.ico",
  "assets/icon.png",
  "assets/icon.icns"
]
```

### 3. Улучшен код main.js
Добавлена функция `getIconPath()` которая правильно определяет путь к иконке в зависимости от платформы и режима (dev/production).

## Проверка иконки

Запустите скрипт для проверки формата иконки:
```bash
node check-icon.js
```

Иконка должна содержать следующие размеры:
- 16x16
- 32x32
- 48x48
- 64x64 (опционально)
- 128x128 (опционально)
- 256x256

## Пересборка приложения

После внесения изменений пересоберите приложение:

```bash
npm run build
```

## Проверка после установки

1. Установите приложение из созданного установщика
2. Проверьте иконку в:
   - Панели задач
   - Рабочем столе (ярлык)
   - Меню Пуск
   - Диспетчере задач
   - Списке установленных программ

## Дополнительные рекомендации

### Для Windows
- Используйте `.ico` файл с несколькими размерами
- Убедитесь что файл не поврежден
- Размер файла должен быть разумным (< 1 MB)

### Для macOS
- Используйте `.icns` файл
- Создайте его с помощью `iconutil` или онлайн конвертера

### Для Linux
- Используйте `.png` файл размером 512x512 или 1024x1024

## Создание иконки из PNG

Если у вас есть только PNG файл, создайте ICO:

```bash
# Установите зависимости (если еще не установлены)
npm install --save-dev to-ico sharp

# Создайте скрипт convert-icon.js и запустите
node convert-icon.js
```

## Troubleshooting

### Иконка все еще не отображается
1. Очистите кэш Windows:
   - Удалите `IconCache.db` из `%localappdata%`
   - Перезагрузите компьютер

2. Проверьте что иконка включена в app.asar:
   ```bash
   # Распакуйте app.asar
   npx asar extract dist/win-unpacked/resources/app.asar extracted
   
   # Проверьте наличие assets/icon.ico
   ls extracted/assets/
   ```

3. Проверьте логи при запуске приложения (должен быть путь к иконке)

### Иконка отображается в dev, но не в production
- Убедитесь что путь к иконке правильный в обоих режимах
- Проверьте что иконка включена в `files` в package.json
- Используйте `__dirname` для построения пути, а не относительные пути
